技术点：

UI：虽然 antd 组件齐全，功能也强大，但很多样式、切换行为个人不喜欢。因此，所有的组件都是自己手写的。 😥

评论：使用的是 Valine 第三方库，引入后创建对象，提供一个 DOM 挂载就行了；除此之外，还可以开启文章阅读量统计，通过连接 leanCode 存储，获取评论和访问量。

Next: 因为我要直接生成静态页面，所以，但凡要用到数据的页面，我都让它静态化，就没有使用 redux 仓库了。

Node: 开始倒是想写写 MySql 数据库，毕竟查询、添加方便很多，但想起之前用 Hexo 搭建的博客直接用的一个个 MarkDown 文件生成页面，很方便，在加上要用的数据也不是很多，所以就直接用 Node 读取 MarkDown 文件，然后解析生成 html 元素，可惜代码高亮弄了一晚上还是没解决。
